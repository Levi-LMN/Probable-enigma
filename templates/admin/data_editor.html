How It Works
1

Export

Download your projects or experiences as a JSON file. Every record includes its database id.

2

Edit

Open the file in VS Code, Notepad++, or any editor. Change any editable field across as many records as you like at once.

3

Re-import

Upload the edited file. Every record is matched by id and updated in place — nothing else is touched.

Tips for Editing JSON

Reordering

Change display_order values (lower = first). You can number them 1, 2, 3 or use gaps like 10, 20, 30.

Bulk hide / show

Set is_visible to false across multiple records at once instead of toggling one by one.

Featured projects

Toggle is_featured on your best work to control what appears on the home page.

Safety

Records with an id that doesn't exist in the DB are silently skipped — you won't accidentally create duplicates.
